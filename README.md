# [![Review Assignment Due Date](https://classroom.github.com/assets/deadline-readme-button-22041afd0340ce965d47ae6ef1cefeee28c7c493a6346c4f15d667ab976d596c.svg)](https://classroom.github.com/a/BbbsAcEO)
Ваша роль влияющая на метрику качества модели:
- вы владелец сети винных магазинов;
- ваша задача приобрести вино для вашей сети качеством не ниже 6 (от 6 включая и выше);
- вы не хотите проникновения в вашу сеть вина качеством от 5 и ниже;
- вы должны обеспечить достаточное разнообразие качественного вина.

Датасет:
- набор физико-химических характеристик вина;
- целевая переменная `quality` - качество вина, зависящее от его физико-химических характеристик.

Разрещенный методы ML:
- подготовка тренировочного набора - любая, но не алгоритмическая;
- препроцессинг - только скалеры;
- алгоритмы ML только из Scikit-Learn:
  1. Любые ближайшие соседи
  2. Любые логистические регрессии
  3. Любые алгоритмы на базе байеского подхода
  4. Машины опроных векторов с любыми ядрами(через неделю после старта)
- любые кросс-валидации из Scikit-Learn; 
- подбор параметров модели:
  1. из Scikit-Learn [любая оптимизация гиперпараметров отсюда](https://scikit-learn.org/stable/api/sklearn.model_selection.html#hyper-parameter-optimizers)
  2. [Optuna](https://optuna.org), а также [отсюда...](https://neptune.ai/blog/hyperparameter-tuning-in-python-complete-guide)
- подбор [порога модели](https://scikit-learn.org/stable/api/sklearn.model_selection.html#post-fit-model-tuning) (если нужно).

Параметры выбранной модели должны быть жестко зафиксированы. Если модель учится долго, то она должна быть обучена и законсервированна. Должен быть получен вектор предсказанных значений целевой переменной для тестового файла.
